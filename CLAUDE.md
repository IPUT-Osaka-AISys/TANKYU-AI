## コーディング規約

### 一般的な原則

- **DRY (Don't Repeat Yourself)**: コードの重複を避ける
- **KISS (Keep It Simple, Stupid)**: シンプルなコードを心がける  
- **YAGNI (You Aren't Gonna Need It)**: 必要になるまで機能を追加しない
- **単一責任の原則**: 各関数やクラスは1つの責任のみを持つべき

### フォーマットとスタイル

- **インデント**: スペース2つを使用
- **命名規則**:
  - クラス: PascalCase (`UserAccount`)
  - 関数/メソッド: camelCase (`getUserData`)
  - 変数: camelCase (`userData`)
  - 定数: SCREAMING_SNAKE_CASE (`MAX_RETRY_COUNT`)
  - プライベートメンバ: 先頭にアンダースコア (`_privateVariable`)
- **コメント**:
  - なぜそのコードが存在するのかを説明（何をしているかではなく）
  - JSDocスタイルのドキュメンテーションコメントを使用

### バージョン管理

- **ブランチ戦略**: Git Flow または GitHub Flow に基づく
  - `main`/`master`: 本番環境用のコード
  - `develop`: 開発用のコード
  - 機能ブランチ: `feature/機能名`
  - 修正ブランチ: `bugfix/問題の概要`
- **コミットメッセージ**: Conventional Commits形式を採用
  ```
  type(scope): タイトル
  
  詳細な説明
  ```

## テストとCI/CD要件

### テスト要件

- **テストカバレッジ**: 80%以上を維持
- **テスト駆動開発 (TDD)**: Red → Green → Refactor のサイクル
- **テストの種類**:
  - 単体テスト: 個々の関数やクラスの動作を検証
  - 統合テスト: コンポーネント間の相互作用を検証
  - E2Eテスト: ユーザーの視点からシステム全体の動作を検証

### CI/CD

- **継続的インテグレーション**: 頻繁にコードを統合し、自動テストを実行
- **継続的デリバリー**: 本番環境へのデプロイを自動化
- **静的解析ツール**: コード品質を確保

## セキュリティとプライバシー要件

### セキュアコーディング

- **入力の検証と無害化**: すべての外部入力を検証
- **パラメータ化されたクエリ**: SQLインジェクション対策
- **XSS対策**: クロスサイトスクリプティング対策
- **CSRF対策**: クロスサイトリクエストフォージェリ対策
- **適切な認証と認可**

### 機密情報の管理

- **シークレット管理**: 環境変数やシークレット管理サービスで管理
- **パスワード**: 常にハッシュ化して保存
- **通信**: 常にTLS/SSLで暗号化
- **最小特権の原則**: アクセス権を適切に設定

### プロジェクト固有のセキュリティ要件

- **データ保護**: 個人を特定できる情報は保存しない
- **HTTPS通信**: すべての通信でHTTPSを使用
- **API認証**: 外部API使用時の適切な認証情報管理
- **ログ管理**: セキュリティログの保存期間は90日間
